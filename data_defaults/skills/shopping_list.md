# רשימת קניות - Shopping List Skill

כאשר המשתמש מבקש לנהל את רשימת הקניות (להוסיף פריטים, למחוק, להציג, למיין או לרוקן את הרשימה), השתמשי בהוראות הבאות:

## מיקום הרשימה
רשימת הקניות נשמרת תמיד בקובץ בשם `SHOPPING_LIST.md` באמצעות הכלים שלך.

## פעולות אפשריות:

1. **הצגת הרשימה (View)**: 
   אם המשתמש שואל מה יש ברשימת הקניות, קראי את הקובץ `SHOPPING_LIST.md` באמצעות הכלי `read_knowledge_file`. אם הוא ריק או לא קיים, עני שהרשימה ריקה.

2. **הוספת פריטים (Add)**:
   - קראי את הרשימה הנוכחית באמצעות `read_knowledge_file`.
   - הוסיפי את הפריטים החדשים לרשימה בסגנון של רשימת תבליטים (כגון `- חלב`).
   - שמרי את הרשימה המעודכנת חזרה לקובץ `SHOPPING_LIST.md` באמצעות הכלי `update_memory`.
   - עדכני את המשתמש שהפריטים נוספו.

3. **מחיקת פריטים / סימון כנקנה (Delete/Complete)**:
   - קראי את הרשימה, מחקי את הפריטים שהמשתמש ביקש (אפשר לזהות באופן חכם גם אם השם לא זהה ב-100%).
   - שמרי את הקובץ המעודכן עם `update_memory`.
   - דווחי למשתמש מה נמחק.

4. **ריקון הרשימה (Empty/Clear)**:
   - אם המשתמש מבקש לרוקן את כל הרשימה או למחוק את כולה, השתמשי ב-`update_memory` כדי להחליף את תוכן הקובץ לטקסט המייצג רשימה ריקה (למשל "*הרשימה ריקה*").
   - דווחי שהרשימה אופסה.

5. **מיון הרשימה (Sort)**:
   - קראי את הרשימה הנוכחית.
   - מייני את הפריטים לפי קטגוריות ומחלקות בסופר.
   - שמרי את הרשימה הממוינת באמצעות `update_memory`.
   - הציגי למשתמש את הרשימה המסודרת.

## כללי אצבע:
- אל תשתמשי בשום כלי אחר עבור רשימת קניות, רק בכלים של הזיכרון (`read_knowledge_file` ו-`update_memory`).
- הרשימה צריכה להיות תמיד קריאה ומסודרת.
- אם הקובץ לא קיים בעת ניסיון קריאה, התייחסי לכך כאילו הרשימה ריקה וצרי אותו כשמוסיפים פריטים.
